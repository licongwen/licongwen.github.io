很久没有写blog了。今天来回顾一下js中的继承。

1、原型继承

缺点：1、需要将子类原型的constructor属性重新指向子类构造函数；
2、若属性为引用类型，则会被所有实例共享,比如：

2、借用构造函数

优点：1、避免了引用类型的属性被实例共享;
2、在子类中可以向父类中传参数；
缺点：1、无法继承原型；
2、每次创建实例时都会调用一次方法。

3、组合继承

组合继承就是将原型方法和构造函数方法结合在一起的方法。

缺点：组合继承是JS最常用的继承模式，但是在组合继承中，父类会被调用两次，一次在子类构造函数内，一次在子类原型继承父类实例。
